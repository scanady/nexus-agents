# Output Format Adaptation Guide

The presentation outline is format-agnostic by design. This reference provides guidance on how to adapt the outline into specific final formats. Include relevant format-specific notes in the outline when the user has specified their target format.

---

## PowerPoint / Slide Deck

**Mapping from outline to slides:**

| Outline Element | Slide Translation |
|----------------|-------------------|
| Meeting Context | Title slide + agenda slide |
| Each Section | 1-3 slides depending on content density |
| Key content bullets | Slide body — aim for 3-5 points per slide maximum |
| Speaker notes | PowerPoint speaker notes field (verbatim usable) |
| Visual/data guidance | Specifies chart type, layout suggestion, or visual metaphor |
| Discussion catalyst | May warrant its own "discussion" slide or be embedded in speaker notes |
| Closing: Next Steps | Dedicated next-steps slide with action table |
| Appendix | Backup slides after the main deck, clearly labeled |

**Slide design principles to embed in the outline:**
- **One message per slide.** Each slide's title should be an action-title (insight statement). If the title is a topic label, the slide is unfocused.
- **6-word titles, not 6-word slides.** Titles carry the insight; the body carries the evidence.
- **Visual over text.** Where the outline says "show trend data," the slide should have a chart, not a paragraph describing the trend.
- **Builds for complex content.** When a section has layered content (e.g., 3 options for a decision), suggest animation builds or sequential slides to control information flow.
- **Consistent structure signals.** Use visual cues to help the audience track where they are: section dividers, progress indicators, or color-coding by objective type.

**Specific PowerPoint notes to include in the outline:**
- Suggested slide count per section
- Whether a section needs a build/animation sequence
- Where a full-bleed image, chart, or diagram would be more effective than text
- Where a "discussion" or "decision" slide should appear (distinct visual treatment)

## HTML Single-Page / Digital Briefing

**Mapping from outline to single-page:**

| Outline Element | HTML Translation |
|----------------|------------------|
| Meeting Context | Header section with metadata (date, audience, objectives) |
| Narrative Arc | Opening summary paragraph or highlighted callout |
| Each Section | H2 section with content organized for scanning (bullets, callouts, tables) |
| Visual/data guidance | Embedded charts, tables, or highlighted stat callouts |
| Discussion catalyst | Callout boxes or highlighted questions |
| Closing: Next Steps | Action table with clear formatting |
| Appendix | Collapsible sections or linked anchor references |

**HTML-specific principles to embed in the outline:**
- **Scannable structure.** Executive readers will scan before they read. Use clear hierarchy: bold key points, use callout boxes for critical data, and keep paragraphs to 2-3 sentences.
- **Data visualization inline.** Where the outline specifies charts or data, note the specific visualization (bar chart, trend line, comparison table) — these render natively in HTML.
- **Progressive disclosure.** Use expandable/collapsible sections for detail that supports the main narrative but shouldn't clutter the primary flow. Good for: methodology details, full data tables, alternative analyses.
- **Navigation.** For longer briefings, include a sticky table of contents or section navigation so readers can jump to the section relevant to them.
- **Mobile-friendly.** If the briefing may be read on mobile (common for executives), note where content should be simplified or reformatted for small screens.

**Specific HTML notes to include in the outline:**
- Where to use callout/highlight boxes vs. inline text
- Where progressive disclosure (expandable sections) would serve the reader
- Which data visualizations should be interactive vs. static
- Whether the page needs print-friendly styling

## Memo / Written Document

**Mapping from outline to memo:**

| Outline Element | Memo Translation |
|----------------|------------------|
| Meeting Context | Header block (To, From, Date, Re, Purpose) |
| Narrative Arc | Executive summary paragraph (first thing after header) |
| Each Section | Numbered or headed sections with narrative prose |
| Key content bullets | Integrated into flowing paragraphs or formatted as numbered points |
| Speaker notes | Become the actual narrative voice of the memo |
| Visual/data guidance | Inline tables, referenced exhibits, or embedded figures |
| Discussion catalyst | Framed as "questions for the group" or "areas requiring input" |
| Closing: Next Steps | Formal next-steps section with action table |
| Appendix | Attached exhibits or appendices referenced in the body |

**Memo-specific principles to embed in the outline:**
- **Executive summary is mandatory.** The first paragraph must contain: the purpose of the memo, the key recommendation or finding, and what is being asked of the reader. An executive who reads only this paragraph should understand the memo's point.
- **Narrative flow, not slide bullets.** Memos use complete sentences and paragraphs. The outline's bullet points become topic sentences and supporting evidence in prose form.
- **Exhibits, not slides.** Data that would be a chart on a slide becomes an exhibit in a memo (Exhibit 1, Exhibit 2) — referenced in the text but presented separately so the narrative isn't interrupted.
- **Precision of language.** Memos are a written record. Language must be more precise than spoken delivery — every claim should be supportable and every recommendation should be clearly stated.
- **Length calibration.** Short memos (1-2 pages): decisions, approvals, brief updates. Medium memos (3-5 pages): analysis readouts, strategy proposals. Long memos (6+ pages): comprehensive research, multi-topic briefs. When in doubt, shorter.

**Specific memo notes to include in the outline:**
- Recommended memo length based on content
- Which sections should be narrative prose vs. structured (tables, lists)
- Where to place exhibits and what each exhibit should contain
- Whether the memo serves as a pre-read, a standalone artifact, or both

## Adapting the Pre-Read by Format

The pre-read brief format is consistent regardless of the final presentation format, but its content emphasis shifts:

| Final Format | Pre-Read Emphasis |
|-------------|-------------------|
| **PowerPoint** | Pre-read carries the context and data the slides won't; set up the "why" so the meeting focuses on the "what now" |
| **HTML single-page** | Pre-read is lighter — the HTML briefing itself is readable. Pre-read focuses on framing questions and setting expectations |
| **Memo** | Pre-read may be unnecessary if the memo is distributed in advance. If used, the pre-read is a very short (half-page) "what to focus on and what we need from you" |

---

## Adapting the Post-Meeting Summary by Format

The post-meeting summary template is format-independent — it captures decisions, actions, and follow-ups regardless of how the presentation was delivered. No format-specific adaptation needed.
