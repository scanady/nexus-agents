# Meeting Minutes Output Format

Use this structure for the final deliverable unless the user explicitly requests a different format.

```markdown
# Meeting Minutes: [Title]

## Meeting Header
- **Title:** [Meeting title]
- **Date:** [Date]
- **Time:** [Time]
- **Location:** [Location / platform]
- **Meeting Type:** [Status meeting / board meeting / project sync / board review / etc.]

## Participants
- [Full Name, Title]
- [Full Name, Title]

## Executive Summary
[2-3 sentence factual summary of the meeting purpose, outcomes, and follow-up direction.]

## Agenda Items

### 1. [Agenda Item]
- Summary of discussion
- Key facts, updates, risks, or questions raised
- Outcome or current status

### 2. [Agenda Item]
- Summary of discussion
- Key facts, updates, risks, or questions raised
- Outcome or current status

## Key Decisions
- **Decision 1:** [Clear statement of what was decided]
- **Decision 2:** [Clear statement of what was decided]

## Action Items

| Task | Owner | Deadline | Status |
|------|-------|----------|--------|
| [Task] | [Full Name] | [Date] | [Open / In Progress / Blocked / Complete] |
| [Task] | [Full Name] | [Date] | [Open / In Progress / Blocked / Complete] |

## Next Steps
- [Follow-up activity]
- [Dependency or milestone]
- **Next Meeting:** [Date / time / location if available]

## Attachments / References
- [Document, link, slide deck, ticket, recording, or `None provided`]

## Assumptions / Gaps
- [Only include when necessary. Note missing, inferred, or unconfirmed details clearly.]
```

## Formatting Rules

- Use clear section headings and bullets for scanability.
- Keep total length appropriate to meeting duration, usually 1-3 pages.
- Present action items in the required table format.
- Use professional business documentation style throughout.
- If a section has no content, state that explicitly instead of omitting it when the omission would create ambiguity.
