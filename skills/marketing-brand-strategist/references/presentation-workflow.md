## 1. How to Present Name Candidates

### Batch Size
Present 5-8 names per round, not 20. More than 8 creates decision paralysis. Fewer than 5 feels like you didn't try hard enough.

### Presentation Format
For each name, provide:
- The domain (with registration link from GoDaddy check)
- Pronunciation guide (only if not obvious)
- One sentence on what makes the name work
- The naming strategy used (metaphor, domain language, portmanteau, etc.)

Group names by strategy or theme, not alphabetically. This helps the user see patterns in their own preferences.

### Context Before Names
Before presenting names, briefly articulate back:
- What you understand about the brand
- What naming strategies you're using and why
- Any constraints (TLD, length, existing brand elements)

This builds trust and ensures alignment before the user evaluates specifics.

### What Not to Do
- Don't present names you haven't checked for domain availability
- Don't present premium or unavailable domains — only standard-price registrations
- Don't explain every name in a paragraph; one line is enough
- Don't apologize for the difficulty of finding .com domains; just do the work
- Don't present names that fail the Phone Test

## 2. Iteration Patterns

### Read the User's Reactions
Pay close attention to which names the user gravitates toward. Their preferences reveal unstated criteria:
- If they like "brewcodex" but not "brewgrimoire," they want authoritative, not mystical
- If they like "elevynt" but not "augmentrik," they value brevity over descriptiveness
- If they keep coming back to a theme (like "alchemy"), lean into it rather than away from it

### Pivot Signals
When the user says something isn't working, diagnose why before generating more:
- "Too complicated" = go shorter, simpler, real-word-based
- "Too generic" = go more specific, use insider language
- "Doesn't feel right" = ask what DOES feel right; get them to name brands they admire
- "I like X but..." = riff on X specifically, not the whole concept

### Progressive Narrowing
A typical naming session goes through 3-5 rounds:
1. Broad exploration (multiple strategies, diverse names)
2. Direction setting (user reacts, you identify patterns)
3. Focused generation (deep dive into the preferred direction)
4. Refinement (variations on top candidates)
5. Final check (trademark search, domain confirmation)

## 3. Domain Checking Workflow

### Check Before You Present
Always check domain availability before showing names to the user. Presenting unavailable names wastes everyone's time and erodes trust.

### Batch Checking Strategy
- Generate 15-20 candidates per concept direction
- Check them all at once via GoDaddy bulk check
- Filter to standard-price available domains
- Present only the best 5-8 from the available set

### Track Results Across Rounds
Keep a mental (or explicit) running tally of:
- Names the user liked (even if domains were unavailable)
- Themes/patterns that resonated
- Names already checked (don't re-check)

### When Domains Are Scarce
If a particular direction yields few available domains:
- Tell the user honestly
- Suggest adjacent directions that might have better availability
- Try longer compound names (two-word .com domains have better availability)
- Consider alternative TLDs (.io, .ai) if appropriate for the brand

## 4. Narrowing and Decision Support

### Side-by-Side Comparison
When the user has 2-3 finalists, present a comparison:
- How each name performs on the evaluation criteria (Phone Test, Memory Test, etc.)
- Practical considerations (length for social handles, email addresses, logo potential)
- How each name supports the brand story

### The Sleep Test
Recommend the user sit with their top 2-3 names for 24-48 hours. Names that stick in your mind without effort are winners. Names you have to keep looking up are losers.

### Trademark Considerations
Remind the user to:
- Search the USPTO trademark database
- Search state trademark databases if US-based
- Google the name to check for existing businesses
- Check social media handle availability
- Note: Claude is not a lawyer and this is not legal advice

### Registration Urgency
Once a name is chosen, register it quickly. Domain availability changes constantly, especially for short/clean names.
